                                      Components

                                      Type: xcd.component.ComponentPackageDescriptor
                                      Location: components.json declarationsREPLACE

                                      Contains component descriptions for all components in a component package directory. It is the result of parsing a "components.json" file.

                                      The package descriptor differs from the other top level models in that it is not subject to composition by the ModelCompositionService but is handled by ComponentRegistryImpl.

                                      It describes component

                                      [Attribute][type][Description]
                                      descriptionString
                                      Description for the component package.
                                      nameString
                                      Name of the top level model.
                                      componentsMap componentName -> ComponentDescriptor
                                      Returns the a map mapping component names to the component descriptors for that component.
                                      xcd.component.ComponentDescriptor: Descriptor for a single component within a component package. It contains a formal description of component properties and their types. It is used by the server side to compose components in the editor, to provide the declared queries and call the declared data providers and by the client-side view-renderer module to render a method producing a reactjs component tree from the JSON model description of the view. @see ComponentPackageDescriptor @see PropDeclaration
                                      [Attribute][type][Description]
                                      childRuleExpression
                                      Completion rule to find all eligible children for this component. The general mechanism is to find all child candidates with this rule and then use a potential parentRule on the child to validate the candidate status.
                                      classesArray
                                      Component classification for this component. The classes are used to determine eligibility for completion or to activate view renderer features.

                                      The class "model-aware" will cause the model and viewModel to be injected into the component.

                                      dataProviderString
                                      Name of spring bean implementing the DataProvider interface to use as an alternate data provider to use for this component. The default is to use QueryDataProvider.
                                      descriptionString
                                      Description for the component.
                                      parentRuleExpression
                                      Validation rule to validate if a component is valid in a parent context. @see #childRule
                                      providesContextString
                                      Defines the symbolic type name of the context provided by this component. Other components can receive that component type either by inheriting it from the first parent to offer any context or by specifying a context type they depend on.
                                      queriesqueryName
                                      Query definitions for this component. The result of the query will show up in the component data block under the defined query name.
                                      queryTransformerString
                                      Bean name of a de.quinscape.exceed.runtime.expression.query.QueryTransformer implementation (Default is "componentQueryTransformer")
                                      varsMap key -> Map
                                      Var definitions for this component
                                      componentPropWizardsMap key -> ComponentPropWizard
                                      Maps the prop name to the a component prop wizard definition for that prop.
                                      xcd.component.ComponentPropWizard: A wizard component definition for the component.
                                      [Attribute][type][Description]
                                      descriptionString
                                      Description for the prop wizard
                                      titleString
                                      Title for the prop wizard component
                                      wizardString
                                      Wizard Component js name.
                                      propTypesMap propName -> PropDeclaration
                                      Maps prop names to PropDeclarations for this component.
                                      xcd.component.PropDeclaration: Declaration for a single property within a {@link ComponentDescriptor}.
                                      [Attribute][type][Description]
                                      clientboolean
                                      true if this prop is visible to the client as a value. Some props are only needed for server operation and / or might contain expression types not available on the client per-se. Setting this to false will cause such an attribute to be only added as a comment.

                                      xcd.config.ComponentConfig: Contains component configurations for the components within an exceed applications. Is embedded in {@link Configuration}

                                      This is were app-level configuration for system components is stored. This can be an actual component model configuration or for another system component. It provides a typed and auto-documented set of configuration options for these (system) components.

                                      If you add multiple properties for the same logical component, you should consider an intermediary container component like {@link BaseTemplateConfig}.
